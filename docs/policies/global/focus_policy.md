# Focus Policy

## Generation Goal

- 전체 기능을 얕고 많이 생성하지 않는다.
- 핵심 기능은 적은 수로도 깊게 생성한다.
- 비핵심 기능은 smoke 수준만 생성하거나 제외한다.
- testcase 수를 늘리는 것보다 리스크 커버리지를 우선한다.

## Core Feature Criteria

다음 중 하나라도 해당하면 핵심 기능으로 본다.

- 사용자 주요 여정의 필수 단계
- 저장, 수정, 삭제, 결제, 예약, 취소, 로그인 관련 기능
- 상태값 전이가 존재하는 기능
- 외부 API, 백엔드, 어드민, DB 연동 기능
- 데이터 불일치 시 장애 영향이 큰 기능
- 권한, 개인정보, 과금, 환불 관련 기능
- 발송, 확정, 실패, 재시도처럼 운영 영향이 큰 기능

## Exclude or Minimize

다음은 기본적으로 상세 생성 대상에서 제외한다.

- 단순 문구 차이
- 정렬, 여백, 색상, 스타일 미세 차이
- 장식성 UI
- 영향도 낮은 tooltip/copy
- 동일 의미의 반복 UI variation
- 상태/권한/데이터/과금 판단과 무관한 정적 화면 요소

## Required Edge Cases for Core Features

핵심 기능은 아래를 우선적으로 포함한다.

- 필수값 누락
- 빈 결과 또는 데이터 없음
- null 또는 malformed 응답
- 상태값별 분기
- 중복 클릭 또는 중복 요청
- 뒤로가기 또는 재진입
- 권한 없음
- 저장 후 반영 시점 지연
- API/UI/DB 정합성
- 실패, 취소, 롤백
- 시작일/종료일, 수량 소진, 0원/1원 이상, N명 같은 경계값

## Per Feature Output Limit

- 핵심 기능: 5~8개 범위에서 리스크 축 중심으로 생성한다.
- 비핵심 기능: 1~2개 smoke 케이스만 생성한다.
- 유사 케이스는 통합한다.
- 같은 검증 의도를 화면명만 바꿔 반복하지 않는다.

## Scope Labels

- `core-deep`: 핵심 기능이며 상태/예외/경계/정합성까지 깊게 생성한다.
- `smoke-only`: 비핵심 기능이며 대표 정상 또는 주요 노출만 확인한다.
- `omitted`: 테스트케이스로 생성하지 않고 제외 사유만 남길 수 있는 세부 항목이다.
