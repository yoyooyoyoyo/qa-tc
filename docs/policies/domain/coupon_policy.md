# 쿠폰 정책

## 최신 업데이트

- 업데이트 일자: 2026-05-02
- 기준 문서: APP v20 쿠폰/예약 정책 참고 메모
- 기준 Figma: `기획서-2026_착한의사App_v.20.0.0`, 쿠폰/예약/어드민 쿠폰관리 화면

## 적용 범위

- APP 병원/검진 상세
- APP 예약/결제
- 예약 접수 실패
- 어드민 쿠폰관리
- 어드민 회원상세 보유쿠폰

## 쿠폰 유형

- 다운로드쿠폰: 사용자가 상품 상세 또는 예약 과정에서 직접 다운로드해 보유할 수 있는 쿠폰
- 코드쿠폰: 쿠폰 코드 입력 또는 외부 지급 등으로 이미 보유 상태가 되는 쿠폰
- 보유 다운로드 쿠폰: 다운로드 완료 후 사용기간 내인 쿠폰
- 미보유 다운로드 쿠폰: 운영 중이나 사용자가 아직 다운로드하지 않은 쿠폰

## 검진상세 쿠폰 카드 노출 정책

- 쿠폰 카드는 비교 대상 쿠폰 중 실제 할인 금액이 가장 큰 쿠폰을 노출한다.
- 비교 대상은 운영 중인 미보유 다운로드 쿠폰, 보유 다운로드 쿠폰, 보유 코드쿠폰이다.
- 노출 우선순위는 금액 높은 순, 사용 종료일 빠른 순, 코드쿠폰 우선 순이다.
- 금액과 종료일이 모두 동일하면 코드쿠폰을 우선 노출한다.
- 비교 가능한 유효 쿠폰이 하나도 없으면 쿠폰 영역 전체를 hide 처리한다.

## 쿠폰 영역 hide 조건

- 다운로드 쿠폰이 미운영이고 코드쿠폰이 미보유인 경우 hide한다.
- 다운로드 쿠폰이 미운영이고 코드쿠폰 발급 수량이 소진된 경우 hide한다.
- 다운로드 쿠폰이 미운영이고 코드쿠폰 발급 기간이 만료된 경우 hide한다.
- 다운로드 쿠폰이 미운영이고 코드쿠폰 사용 기간이 만료된 경우 hide한다.
- 코드쿠폰이 미보유이고 다운로드 쿠폰 발급 수량이 소진된 경우 hide한다.
- 코드쿠폰이 미보유이고 다운로드 쿠폰 발급 기간이 만료된 경우 hide한다.
- 코드쿠폰이 미보유이고 다운로드 쿠폰 사용 기간이 만료된 경우 hide한다.

## 예약/결제 쿠폰 사용 정책

- 예약/결제 화면에는 쿠폰사용, 사용가능쿠폰없음, 쿠폰있으나 사용하지 않음, 쿠폰사용툴팁 상태를 분리한다.
- 쿠폰 사용 영역은 할인혜택 또는 멤버십 사용 영역 안에서 쿠폰명, 혜택 태그, 도움말 아이콘, 쿠폰 적용 결과를 노출한다.
- 사용 가능한 쿠폰이 없어도 결제금액과 결제 상세 영역은 유지하고, 쿠폰 영역 상태만 달라지는지 확인한다.
- 쿠폰이 있으나 사용하지 않은 경우 할인금액과 최종 결제금액이 쿠폰사용 케이스와 다르게 반영되는지 확인한다.

## 쿠폰 실패 정책

- 쿠폰 문제로 예약 접수 실패 화면이 분기될 수 있다.
- 쿠폰 만료 사유는 `사유 : 쿠폰 만료`로 노출한다.
- 쿠폰 만료 안내 문구는 `사용하신 쿠폰의 사용 기간이 끝나 사용할 수 없어요. 다른 쿠폰으로 변경 후 다시 예약해 주세요.`이다.
- 쿠폰 중단 사유는 `사유 : 쿠폰 중단`으로 노출한다.
- 쿠폰 중단 안내 문구는 `사용하신 쿠폰이 운영상 중단되어 사용할 수 없어요. 다른 쿠폰으로 변경 후 다시 예약해 주세요.`이다.
- 쿠폰 오류 사유는 `사유 : 쿠폰 오류`로 노출한다.
- 쿠폰 오류 안내 문구는 `일시적인 오류로 쿠폰을 사용할 수 없어요. 잠시 후 다시 예약해 주세요.`이다.

## 어드민 쿠폰 상태

- 쿠폰 상태 축은 대기, 발행시작, 운영중, 중지, 종료이다.
- 미래 날짜 발행은 대기 상태가 된다.
- 즉시 발행은 발행시작을 거쳐 운영중이 된다.
- 관리자가 직접 설정하면 중지 상태로 전환될 수 있다.
- 발행기간 종료, 쿠폰 사용기간 종료, 발행수량 완전 소진 시 종료 상태가 된다.
- 발급기간 종료 후 사용기간 내인 쿠폰은 운영중으로 표기할 수 있다.

## TC 작성 시 분리 기준

- 쿠폰 유형: 다운로드쿠폰, 코드쿠폰
- 쿠폰 보유 상태: 미보유, 보유중, 사용완료
- 쿠폰 운영 상태: 대기, 발행시작, 운영중, 중지, 종료

- 실패 사유: 쿠폰 만료, 쿠폰 중단, 쿠폰 오류
- 사용자 유형: B2C, B2B, B2B2C, 멤버십, 비멤버십
- 화면: 검진 상세, 예약/결제, 예약내역, 어드민 쿠폰관리, 어드민 회원상세
