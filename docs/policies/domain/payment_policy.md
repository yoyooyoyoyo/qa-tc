## 결제금액 구성

- 결제금액은 예상요금, 기본 비용, 추가비용, 할인금액, 부가세, 착한의사 케어비, 최종 결제금액으로 구성될 수 있다.
- 최종 결제금액은 강조 금액으로 노출한다.
- 추가비용은 접힘/펼침 가능한 목록 형태로 노출될 수 있다.
- 멤버십 케이스에서는 착한의사 케어비 할인, 건강검진비 할인이 노출될 수 있다.
- B2B 가격 노출 케이스에서는 검진지원금 영역을 별도로 확인한다.

## 가격 산식 기준

- P는 상품금액이다.
- m은 마진율 또는 마진금액 계산 요소로 본다.
- E는 추가비용이다.
- S는 착한의사 케어비이며, 0원일 경우 면제 표기 케이스가 있다.
- D는 할인금액이며, 0원일 경우 영역 hide 케이스가 있다.
- G는 지원금이며, 지원금이 금액보다 클 경우 사용자 결제금액은 0원으로 표기한다.
- v는 부가세율로 보며, 실제 API 필드 매핑은 확인이 필요하다.

## 툴팁/안내 문구

- 예상요금 툴팁은 상품 마진가가 없을 경우 hide한다.
- 예상요금 툴팁 문구는 `최대 49% 할인이 적용된 검진상품을 제공하기 위한 서비스 이용요금이 포함됩니다.`이다.
- 착한의사 케어비 툴팁 문구는 `검진 여정을 세심하게 케어하기 위해 예약대행, 검진 AI 서비스, 의료진 상담, 결과조회 등 검진 전후 건강관리를 지원합니다.`이다.
- 부가세 툴팁 문구는 `건강검진 여정을 케어하고, 검진 결과를 바탕으로 비의료 건강관리 서비스를 제공하는 착한의사는 의료기관이 아닌 일반기업으로 할인금액을 제외한 결제금액의 10%인 부가세를 국세청에 납부합니다.`이다.

## 결제방법

- 결제방법 후보는 카드결제, 계좌이체이다.
- 현장결제 안내 문구는 `검진비용만 병원에 방문해서 결제합니다.`로 확인된다.
- 결제방법 라디오 선택 상태와 금액 영역의 조합을 함께 확인한다.
- 기존 주문건 중 합계가 1원 이상인 경우 카드결제내역을 hide할 수 있다.
- 기존 주문건 중 합계가 0원인 경우 결제방법과 카드결제내역을 hide할 수 있다.

## B2B 지원금/가격 정책

- B2B 기업검진은 가격노출과 가격미노출 케이스를 분리한다.
- 가격노출 케이스는 검진지원금 사용 여부, 즉시결제/현장결제, 결제금액 0원/1원 이상 여부를 조합한다.
- 가격미노출 케이스는 건강지원금 사용/소진 여부, 즉시결제/현장결제, 결제금액 0원/1원 이상 여부를 조합한다.
- 방문 결제금액이 있는 케이스는 금액 하단에 `병원 방문시 결제` 보조 문구가 노출될 수 있다.

## 0원 결제 정책

- 결제금액이 0원인 경우 결제방법, 카드결제내역, 영수증 노출 여부를 별도로 확인한다.
- 0원 결제는 영수증 확인 불가 토스트가 노출될 수 있다.
- 부가세, 할인금액, 케어비는 0원 산출 시 hide, 0원 표기, 면제 표기 중 항목별 정책을 확인한다.

## TC 작성 시 분리 기준

- 사용자 유형: B2C, B2B
- 검진 유형: 종합검진, 채용검진, 검진확인서, 기업검진
- 결제 방식: 즉시결제, 현장결제, 카드결제, 계좌이체
- 금액 조건: 0원, 1원 이상
- 할인 조건: 쿠폰 사용, 쿠폰 미사용, 멤버십 사용, 멤버십 미사용
- 지원금 조건: 지원금 사용, 지원금 소진, 지원금 선적용, 지원금 후적용
- 결제정보 유형: 신규 주문건, 기존 주문건

## QA 확인 필요

- 가격 factor P, m, E, S, D, G, v의 정확한 API 필드 매핑을 확인한다.
- 부가세 계산 기준과 화면 행 매핑을 확인한다.
- 0원 산출 시 각 항목이 hide인지 0원/면제 표기인지 확인한다.
- B2C 화면의 검진지원금 관련 라벨 의미를 확인한다.
