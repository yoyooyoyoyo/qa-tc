# 예약 정책

## 최신 업데이트

- 업데이트 일자: 2026-05-02
- 기준 문서: APP v20 쿠폰/예약 정책 참고 메모, 예약내역 정책 참고 메모
- 기준 Figma: APP 예약/결제, WEB 예약내역, 예약내역 상세

## 적용 범위

- APP 예약/결제
- APP 예약확정
- WEB 예약내역
- WEB 예약내역 상세
- 어드민 예약 항목

## 예약 상태

- 확정대기: 예약 요청은 접수되었으나 검진일 또는 병원 확정이 완료되지 않은 상태
- 예약확정: 병원 또는 검진일이 확정된 상태
- 검진완료: 검진 방문이 완료된 상태
- 결과등록: 검진 결과 또는 결과 관련 후속 상태가 등록된 상태
- 취소요청: 사용자가 예약 취소를 요청한 상태
- 취소완료: 예약 취소 처리가 완료된 상태
- 미수검: 예약확정 이후 검진을 받지 않은 상태

## 예약/결제 화면 구성

- 예약 화면은 예약카드+날짜, 방문자정보, 방문자특이사항, 멤버십/할인혜택, 결제정보, 결제방법, 하단 버튼으로 구성한다.
- 방문자정보 영역에는 검진 예약내역 제목과 알림 배너가 노출될 수 있다.
- 방문자특이사항 영역에는 수정사항 입력 또는 확인 영역이 있다.
- 예약확정 선결제 화면은 진행 상태, 예약카드+날짜, 방문자정보, 방문자특이사항, 할인혜택, 결제정보, 결제방법, 안심케어, 하단 버튼/주의사항으로 구성한다.

## 예약내역 기본 노출

- 예약내역카드는 병원명, 상품명, 상태 배지, 예약번호, 방문자, 검진일 또는 희망일 정보를 기본으로 노출한다.
- 검진일이 확정되지 않은 케이스는 1지망, 2지망 날짜 형태로 노출될 수 있다.
- 카드 우측 화살표는 상세 진입 액션으로 본다.
- 서비스 이용 동의 없음 상태가 예약내역/결제정보 노출에 영향을 주는지 확인한다.

## 예약 변경/취소

- 예약취소는 확정대기 또는 예약확정 계열에서 노출될 수 있으며, 상태와 검진 유형에 따라 노출 여부가 달라진다.
- 변경은 예약 날짜 변경 액션으로 보며, B2B 특수 검진에서는 별도 변경 모달과 GA 확인이 필요할 수 있다.
- 취소요청과 취소완료는 이전 상태가 확정대기인지 예약확정인지에 따라 분기한다.

## 하단 버튼 정책

- 버튼은 상태, 결제 방식, 검진 유형에 따라 노출/미노출 또는 활성/비활성이 달라진다.
- 버튼 상태는 없음, 활성, 비활성, 라인 버튼, 강조 버튼, 아이콘 버튼, 2버튼 병렬 노출로 분리한다.
- 예약내역카드 버튼 정책은 종합검진/채용검진/기업검진(종합/채용) 그룹과 국가검진/기업검진(특수/국가) 그룹을 분리한다.
- 예약내역 상세 하단 버튼은 종합검진/채용검진/기업검진(종합/채용), 국가검진/기업검진(국가), 기업검진(특수) 그룹을 분리한다.

## TC 작성 시 분리 기준

- 사용자 유형: B2C, B2B
- 검진 유형: 종합검진, 채용검진, 국가검진, 기업검진, 기업검진(국가), 기업검진(특수), 검진확인서
- 예약 상태: 확정대기, 예약확정, 검진완료, 결과등록, 취소요청, 취소완료, 미수검
- 상태 부가 조건: 평점 작성 전/후, 기간만료 전/후, 이전 상태
- 결제 방식: 즉시결제, 현장결제, 카드결제, 계좌이체
- 액션: 변경, 예약취소, 결제 영수증, 삭제하기, 1:1 주치의 상담, 평점남기기

## QA 확인 필요

- 예약 상태별 버튼 표의 O, X, O(disable), X(disable)의 정확한 의미를 최신 Figma 기준으로 확인한다.
- 검진완료와 결과등록은 평점 작성 여부 또는 기간만료 여부에 따른 버튼 분기를 확인한다.
- 취소요청과 취소완료의 이전 상태 기준 분기를 확인한다.
- B2B 특수/국가 검진의 버튼 노출과 GA 이벤트 확인 범위를 확인한다.
