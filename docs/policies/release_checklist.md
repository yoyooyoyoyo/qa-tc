# Release Checklist

## Smoke Test

- 로그인/로그아웃 주요 플로우가 정상 동작한다.
- 주요 메뉴 진입이 가능하다.
- 주요 목록/상세 화면이 정상 로딩된다.
- 핵심 API 응답 오류가 없다.

## Regression Test

- 기존 정상 플로우가 신규 변경으로 깨지지 않았는지 확인한다.
- 권한별 접근 정책이 유지되는지 확인한다.
- 기존 상태값의 노출/수정 가능 여부가 유지되는지 확인한다.
- 기존 어드민 관리 기능이 신규 변경과 충돌하지 않는지 확인한다.

## Data Validation

- 필수값 누락 시 저장/진행이 차단된다.
- 잘못된 형식 입력 시 오류 메시지가 노출된다.
- 중복 데이터 정책이 적용된다.
- 삭제/취소/수정 시 관련 데이터 상태가 일관되게 변경된다.

## Platform Check

- WEB과 APP 요구사항이 다르면 플랫폼별 테스트케이스를 분리한다.
- 반응형 또는 모바일 화면 정책이 있으면 별도 검증한다.
- 브라우저 또는 OS 제한이 있으면 precondition에 명시한다.

## Release Decision

- Critical 결함이 남아 있으면 배포 불가로 판단한다.
- High 우선순위 결함은 배포 전 수정 여부를 확인한다.
- blocked 테스트가 있으면 원인과 영향 범위를 기록한다.
