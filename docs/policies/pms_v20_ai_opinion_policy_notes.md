## 종합소견 생성 상태값 플로우 (node `757-26148`)

### 상태값 종류

| 상태 | 색상 | 설명 |
|---|---|---|
| 등록완료 | 보라 | 검진결과 등록 완료 후 최초 상태 |
| 소견생성중 | 보라 | AI 최초 소견 생성 진행 중 |
| 소견생성 | 보라 | AI 소견 생성 완료 |
| 소견확정 | 보라 | 관리자가 소견을 최종 확정한 상태 |
| 생성실패 | 보라 | AI 소견 생성 실패 |

### 상태 전이 플로우

```
종합소견 생성 (시작)
    │ AI 서버로 검진결과 전송
    ▼
검진결과 등록
    │
    ▼
등록완료
    │ AI 최초 소견 생성 진행
    ▼
소견생성중 ─────────────────────── AI가 소견 생성을 실패 ──▶ 생성실패
    │                                                              │
    │ 소견 생성이 완료                              소견재생성 진행 시 ← 소견재생성
    ▼                                           (상태값 소견생성중으로 변경)
소견생성
    │ 소견 생성 후, 관리자 유저가 수정 가능
    ▼
AI 소견 수정하기
    │ 수정한 내용에 대해서 종합소견으로 확정
    ▼
소견확정
    │ 소견 확정 이후에도 재생성 설정 가능
    │
    ├─▶ (재생성 시) 소견생성중으로 복귀
    │   소확정 이후, 관리자 유저가 소견 수정 가능
    │
    ▼
종합소견에 적용 (종합소견 CH 673)
```

- **검진결과 수정** 시: 기존 검진결과값 수정 → AI 종합소견 재생성 필요 (2차 개발분)

## 추적관찰 보고서 목록 TO-BE (node `849-22275`)

### 기능 개요

- 병원 행정직원이 추적관찰 보고서를 발송하여 수검자가 다시 착한의사에 유입할 수 있도록 제공
- **발송 시 건당 10,000원 과금 (부가세 별도)** — 과금 안내 필수
- 발송 기능에 집중하도록 UI 개선

### 과금 안내 배너 (고정 노출)

> "추적관찰 보고서를 통해 모바일 검진결과를 발송할 수 있으며, 발송 시 1건당 10,000원(부가세 별도)이 과금됩니다."

### 리스트 컬럼 구성 (`2`)

| 컬럼 | 비고 |
|---|---|
| 예약방식 / 검진타입 / 병원 / 이름 / 생년월일 / 성별 / 검진완료일 / 정보제공동의 | 대상자 정보 |
| 상태 | 8가지 상태값 |
| 등록/수정일시 | 추후 등록일시/수정일시 분리 예정 |
| 발송(예정)일 | 발송이력 저장 예정 |
| 작성자 | 등록/수정/발송 담당자 업데이트 (현재: 행위자로 업데이트) |

### 상태값 정의 (`2-1`)

```
등록완료 / 보고서생성중 / 보고서생성 / 생성실패 / 발송대기 / 발송성공 / 발송실패 / 발송취소
```
- 각 상태값 클릭 시 해당 상태만 리스트업

### 발송 버튼 (`2-2`)

- 클릭 시 [발송일 안내] 모달 노출

## 추적관찰 보고서 목록 > 과금안내 TO-BE (node `865-10054`)

### 발송 플로우 (4단계)

```
① 발송 버튼 클릭 (체크박스 선택 후 활성화)
    ↓
② 발송일 선택 모달 노출 → 발송 예약 버튼 클릭 → 지정일 오전 11시로 발송예정일 업데이트
    ↓
③ 과금 안내 모달 노출 → 발송 대상자 / 예정일 / 예상비용 확인
    ↓
④ 발송 예약 성공/실패 토스트바 노출
```

### 발송 버튼 활성화 조건

- 체크박스 선택 시 발송 버튼 활성화
- **체크박스 disable 상태**: 등록완료 / 보고서 생성중 / 생성실패 (이너핑크 미생성 → 발송 불가)
- **발송 가능 상태**: 보고서 생성 이후 (`보고서 생성` 상태부터)

### 발송일 선택 모달 (`1-1`)

- 모달 내 [행동 안내 배너] 추가 (동작은 기존과 동일)
- 문구: `발송 일시를 선택해 주세요. 선택한 일정에 맞춰 추적관찰 보고서가 발송됩니다.`
- 예시: `2024-06-21(월) 오전 11시 발송예약`

### 발송 모달 하단 버튼 (`1-2`)

- 버튼명 변경: **[발송] → [발송 예약]**
- 발송 시간: 지정일 **오전 11시** 고정

### 과금 안내 모달 (`1-3`)

- 모달 내 표시 내용:
  - 발송 대상자: N명
  - 발송예정일: yyyy-mm-dd
  - 예상 비용: N원 (발송 건 당 10,000원, 부가세 별도)
  - 전송 방식: 카카오 알림톡 또는 문자(SMS)
- **비과세 금액으로 노출**
- 주의 문구: `발송성공 후에는 과금 내역은 취소 및 환불이 불가합니다.`

### 발송 예약 결과 토스트바 (`1-4`)

| 결과 | 토스트 문구 |
|---|---|
| 성공 | `검진결과지 발송 예약을 성공하였습니다.` |
| 실패 | `검진결과지 발송 예약을 실패하였습니다. 다시 한 번 시도해주세요.` |

## 추적관찰 보고서 상세 > 미리보기 (node `894-10075`)

### 미리보기 이관

- 기존 종합소견 생성의 [미리보기] → **추적관찰 보고서**로 이관
- 상단 명칭 변경: `[미리보기]` → **`[추적관찰 보고서 미리보기]`**

### 미리보기 가능/불가 상태

| 가능 여부 | 상태 |
|---|---|
| **미리보기 가능** | 보고서 생성 / 발송대기 / 발송성공 / 발송실패 / 발송취소 |
| **미리보기 불가** | 보고서 생성중 / 생성실패 (AI 검진요약 생성 전) |

### 미리보기 ON/OFF

- **ON**: 오른쪽 패널에 추적관찰 보고서 미리보기 노출
- **OFF**: 화면에 패널 날개버튼 노출 → 클릭 시 오른쪽 패널 오픈
