## 범위

- 주요 케이스는 B2C 예약내역, B2B 가격 노출 예약내역, 멤버십 선결제, 현장결제/즉시결제 조합이다.

## 상태/표기 규칙

- `hide`: 기존에 노출되던 항목이 비노출로 바뀐 케이스를 의미한다.
- `없음/있음`: 처음에는 항목이 없다가 새로 생긴 케이스를 의미한다.
- B2B 케이스는 지원금 여부, 결제방법, 버튼 노출 조건을 필수로 확인한다.
- 예약 상태 예시는 `확정대기`, `검진완료`가 확인된다.
- 예약내역카드 버튼 정책에서 확인되는 상태 축은 `확정대기`, `예약확정`, `검진완료`, `결과등록`, `취소요청`, `취소완료`, `미수검`이다.
- `검진완료`와 `결과등록`은 평점 작성 전/후 또는 기간만료 전/후에 따라 버튼 노출이 달라질 수 있다.
- `취소요청`과 `취소완료`는 이전 상태가 `확정대기`인지 `예약확정`인지에 따라 분기된다.
- `미수검`은 `예약확정` 이후의 상태로 확인된다.

## 예약정보 영역

- 예약정보카드는 예약 상태, 예약/검진 정보, 병원/검진명, 예약번호 등 기본 정보를 노출하는 기준 영역이다.
- 검진명 영역은 단일 검진명, 국가검진 패키지명, 옵션형 선택 항목이 함께 고려된다.
- 예약내역 상세에서는 사용자 입력/확인 영역으로 보이는 항목들이 존재한다.
- 고객 정보성 입력 영역은 이름/연락처/기타 요청 또는 문진성 정보처럼 항목별 노출 차이가 있을 수 있으므로, 실제 TC 작성 시 필드명과 필수 여부를 Figma에서 다시 확인한다.
- 예약내역카드는 병원명, 상품명, 상태 배지, 예약번호, 방문자, 검진일 또는 희망일 정보를 기본으로 노출한다.
- 예약/취소요청 계열에서 검진일이 확정되지 않은 케이스는 `1지망`, `2지망` 날짜 형태로 노출될 수 있다.
- 카드 우측 화살표는 상세 진입 액션으로 보이며, 버튼 노출 상태와 별개로 카드 상세 이동 가능 여부를 확인한다.
- B2C 멤버십 현장결제 확정대기 케이스와 B2B 방문결제 취소요청 케이스가 예약내역 표시 대표 케이스로 확인된다.
- `서비스 이용 동의 없음` 상태가 표시 매트릭스에 포함되어 있으므로, 예약내역/결제정보 노출과 서비스 이용 동의 여부의 관계를 확인한다.

## 결제금액 영역

- 결제금액은 케이스에 따라 `예상요금`, `기본 비용`, `추가비용`, `할인금액`, `부가세`, `착한의사 케어비`, `최종 결제금액`으로 구성된다.
- `최종 결제금액`은 강조 금액으로 노출된다.
- `부가세`와 `착한의사 케어비`에는 도움말 아이콘이 붙는 케이스가 있다.
- 추가비용은 접힘/펼침 가능한 목록 형태로 보이며, 항목 예시는 `위내시경 수면비용`이다.
- 멤버십 케이스에서는 `착한의사 케어비 할인`, `건강검진비 10% 할인`이 있는 것으로 표기되어 있다.
- B2B 가격 노출 케이스에서는 `검진지원금` 영역이 별도로 존재한다.
- 가격정책 UI 변경안 기준으로 `예상요금`, `부가세`, `착한의사 케어비`에는 툴팁/안내 아이콘 노출 정책이 있다.
- `예상요금` 툴팁은 상품 마진가가 없을 경우 hide한다.
- B2C 표시 케이스에도 `검진지원금 본인`으로 보이는 영역이 확인되므로, 실제 의미가 B2C 지원금인지 사용자 부담금 라벨인지 TC 작성 전에 확인한다.

## 가격정책 산식/툴팁

- 가격 factor는 `P`, `m`, `E`, `S`, `D`, `G`, `v` 조합으로 관리된다.
- `P*(1+m)`은 `상품금액 + 마진금액`으로 정의되어 있다.
- `E`는 추가비용이다.
- `S`는 착한의사 케어비이며, 0원일 경우 `면제` 표기 케이스가 있다.
- `D`는 할인금액이며, 0원일 경우 영역 hide 케이스가 있다.
- `G`는 지원금으로 보이며, 지원금이 금액보다 클 경우 0원으로 표기한다.
- `v`는 부가세율로 보이며, 실제 적용 기준은 TC 작성 전에 확인한다.
- B2C 종합/채용 기준 산식 예시는 `P*(1+m)+S`, `P*(1+m)+E`, `(P*(1+m)+S+E-D)*v`, `(P*(1+m)+S+E-D)*(1+v)`이다.
- 지원금 적용 산식 예시는 `P*(1+m)-G`, `P*(1+m)-G+E`, `(P*(1+m)-G+E)*v`, `(P*(1+m)-G+E)*(1+v)`이다.
- 목록/상세 등 결제화면 외 모든 영역과 결제화면은 가격 표시 방식이 다를 수 있다.
- 선결제/방문결제, 지원금 선적용/후적용, 멤버십 사용/미사용 케이스를 분리해서 확인한다.
- `예상요금` 툴팁 문구: `최대 49% 할인이 적용된 검진상품을 제공하기 위한 서비스 이용요금이 포함됩니다.`
- `착한의사 케어비` 툴팁 문구: `검진 여정을 세심하게 케어하기 위해 예약대행, 검진 AI 서비스, 의료진 상담, 결과조회 등 검진 전후 건강관리를 지원합니다.`
- `부가세` 툴팁 문구: `건강검진 여정을 케어하고, 검진 결과를 바탕으로 비의료 건강관리 서비스를 제공하는 착한의사는 의료기관이 아닌 일반기업으로 할인금액을 제외한 결제금액의 10%인 부가세를 국세청에 납부합니다.`
- 부가세 안내 기준은 할인금액을 제외한 결제금액의 10%로 설명되어 있다.
- 0원 산출 시 `부가세`, `할인금액`, `케어비`가 각각 hide인지 0원/면제 표기인지 항목별로 분기된다.

## 결제정보 상세 정책

- 결제정보 신규 주문건 기준으로 B2C 종합검진, B2C 검진확인서, B2B 기업검진 케이스가 분리되어 있다.
- B2C 종합검진은 멤버십 적용 여부에 따라 `멤버십O`, `멤버십X` 케이스로 나뉜다.
- 멤버십O 케이스는 `현장결제 1원 이상 결제`, `즉시결제 1원 이상 결제`, `멤버십O(선택안함 or 잔여 0회) + 즉시결제 1원 이상 결제`가 확인된다.
- 멤버십X 케이스는 `현장결제 1원 이상 결제`, `즉시결제 1원 이상 결제`가 확인된다.
- B2C 검진확인서 결제정보에는 `결제방법`, `결제내역`, `카드결제내역`이 별도 섹션으로 존재한다.
- 결제내역 예시 구성은 `예상요금`, `할인금액`, `부가세`, `합계`이다.
- 카드결제내역은 카드 결제 관련 상세 정보 영역이며, 기존 주문건에서는 hide 대상이 될 수 있다.
- 기존 주문건 중 합계가 1원 이상인 경우 `카드결제내역`을 hide한다.
- 기존 주문건 중 합계가 0원인 경우 `결제방법(카드결제/계좌이체)`과 `카드결제내역`을 hide한다.

## 결제방법 영역

- 결제방법 후보는 `카드결제`, `계좌이체`가 확인된다.
- 현장결제 안내 블록이 별도 컴포넌트로 노출되는 케이스가 있다.
- B2B는 결제방법 노출 여부가 필수 확인 대상이다.
- 결제방법 라디오 선택 상태와 금액 영역의 조합을 함께 확인해야 한다.
- 현장결제 안내 문구는 `검진비용만 병원에 방문해서 결제합니다.`로 확인된다.

## B2B 지원금/가격 정책

- B2B 기업검진은 `가격노출`과 `가격미노출` 케이스가 분리되어 있다.
- 가격노출 케이스는 검진지원금 사용 여부, 즉시결제/현장결제, 결제금액 0원/1원 이상 여부를 조합해 확인한다.
- 가격노출 케이스 예시는 다음과 같다.
- `검진지원금 사용 + 현장결제 + 1원 이상 결제`
- `검진지원금 사용 + 즉시결제 + 1원 이상 결제`
- `검진지원금 사용 + 현장결제 + 0원 결제`
- `검진지원금 사용 + 즉시결제 + 0원 결제`
- `검진지원금 소진(잔여없음) + 즉시결제 + 1원 이상 결제`
- 가격미노출 케이스는 건강지원금 사용/소진 여부, 즉시결제/현장결제, 결제금액 0원/1원 이상 여부를 조합해 확인한다.
- 가격미노출 케이스 예시는 다음과 같다.
- `건강지원금 사용(-10만원) + 즉시결제 + 1원 이상 결제`
- `건강지원금 사용(-25만원) + 즉시결제 + 0원 결제`
- `건강지원금 사용(-10만원) + 현장결제 + 1원 이상 결제`
- `건강지원금 소진(잔여없음) + 현장결제 + 1원 이상 결제`
- B2B 결제정보에는 `검진지원금` 영역, `예상요금`, `기본 비용`, `추가비용`, `즉시 결제금액` 또는 `방문 결제금액`이 조합별로 달라질 수 있다.
- 방문 결제금액이 있는 케이스는 금액 하단에 `병원 방문시 결제` 보조 문구가 노출된다.
- B2B 새 예약내역 표시 매트릭스에는 `예약내역별 B2B`, `예약플로우 > 예약내역 (결제)`, `내정보 > 예약내역` 흐름이 함께 정리되어 있다.
- B2B 표시 매트릭스 라벨 예시는 `C-4(검진지원금)`, `C-5(부가세)`, `D-1(결제방법)`이다.
- B2B 방문결제 취소요청 케이스에는 `가격미노출_현장결제`와 비활성 하단 버튼(`h48/disable`) 케이스가 확인된다.

## 버튼/하단 액션

- 버튼은 상태, 결제 방식, 검진 유형에 따라 노출/비노출 또는 활성/비활성이 달라진다.
- 설계서에는 `btn활성화`, `BTN 없음`, `disable/icon`, `line/icon` 케이스가 확인된다.
- 국가검진, 기업검진, 기업검진(국가), 기업검진(특수)별 하단 버튼 정책이 분리되어 있으므로 TC 작성 시 검진 유형별로 나눠 확인한다.
- B2B는 버튼 노출 조건이 필수 확인 대상이다.
- 예약내역카드 버튼 정책은 크게 두 그룹으로 분리된다.
- A 그룹: 종합검진, 채용검진, 기업검진(종합/채용)
- B 그룹: 국가검진, 기업검진(특수/국가)
- A 그룹과 B 그룹은 같은 예약 상태라도 버튼 노출/활성 정책이 다를 수 있다.
- 확인된 카드 버튼 예시는 `1:1 주치의 상담`이다.
- `1:1 주치의 상담`은 상태/검진유형에 따라 활성 버튼 또는 비활성 버튼으로 노출된다.
- 일부 상태에서는 2개 버튼이 병렬로 노출되는 케이스가 있다.
- 국가검진/기업검진(특수,국가) 그룹에는 단일 강조 버튼이 노출되는 케이스가 확인된다.
- 표기상 `O(disable)`은 버튼이 보이지만 비활성 상태임을 의미한다.
- 표기상 `X(disable)`은 해당 버튼/동작이 없거나 사용할 수 없는 비활성 정책으로 보이며, TC 작성 전 원본 표의 의미를 재확인한다.

## 예약내역 상세 하단 버튼 정책

- 예약내역 상세 하단 버튼 정책은 세 그룹으로 분리된다.
- A 그룹: 종합검진, 채용검진, 기업검진(종합/채용)
- B 그룹: 국가검진, 기업검진(국가)
- C 그룹: 기업검진(특수)
- 상세 하단 버튼 정책에서 확인되는 상태 축은 `확정대기`, `예약확정`, `검진완료`, `결과등록`, `취소요청`, `취소완료`, `미수검`이다.
- 확인된 버튼/액션 예시는 `변경`, `예약취소`, `결제 영수증`, `삭제하기`이다.
- `변경`은 예약 날짜 변경 액션으로 보이며, GA 메모에 `내정보_날짜변경_B2B특수 > 검진일변경모달_변경하기`가 언급되어 있다.
- `예약취소`는 확정대기/예약확정 계열에서 노출될 수 있으며, 상태와 검진 유형에 따라 노출 여부가 달라진다.
- `결제 영수증`은 영수증 안내 영역 아래에 노출되는 버튼이다.
- `삭제하기`는 취소완료 또는 미수검 등 종료성 상태에서 노출되는 케이스가 있으며, `disable/icon`과 `line/icon` 상태가 모두 확인된다.
- 국가검진, 기업검진(국가) 그룹에서는 일부 상태에 `BTN 없음`이 명시되어 있다.
- 상세 하단 버튼 표에서 `결과등록(미작성, 미등록)`처럼 결과등록 상태가 세부 조건으로 분리될 수 있으므로 TC 작성 시 결과/평점 작성 여부를 함께 둔다.
- GA 메모에 `내정보_예약내역_B2B특수 > 평점남기기, 결제영수증`이 언급되어 있어, 기업검진(특수) 상세 버튼은 GA 이벤트까지 확인 대상이다.

## 안내/영수증/확인서

- 취소 안내: 건강검진이 확정되면 검진일 기준 3일 전까지 취소할 수 있으며, 취소 수수료는 발생하지 않는다는 문구가 확인된다.
- 건강검진 확인서: 기관/단체 제출용 건강검진 확인서는 해당 병원에서 발급받는다는 안내가 확인된다.
- 진료비 영수증: 건강검진은 진단 및 치료 목적의 검사가 아니므로 병원에 직접 문의하라는 안내가 확인된다.
- 별도 영수증 안내 문구로 `영수증에는 검진일이 표시되지 않아요. 검진일은 예약내역의 [예약 날짜]를 확인해주세요. 병원에서 결제한 영수증은 병원에 문의해주세요.`가 확인된다.
- 결제 영수증 또는 검진 확인서가 비활성/불가한 상황에서는 토스트 문구를 expected result로 확인한다.
- 확인된 토스트 문구는 `검진완료 후 확인 가능해요!`, `현장결제는 병원에서 발급 가능해요!`, `과거 예약건은 영수증 확인이 불가해요.`, `0원 결제는 영수증 확인이 불가해요.`이다.
- 즉시결제 케이스에는 결제 영수증 확인 가능 여부를 별도로 확인한다.
- 현장결제 케이스는 서비스 내 영수증 대신 병원 발급 안내 토스트가 노출될 수 있다.
- 0원 결제 케이스는 영수증 확인 불가 토스트가 노출될 수 있다.
- 과거 예약건은 영수증 확인 불가 토스트가 노출될 수 있다.

## TC 작성 시 분리 기준

- 사용자 유형: B2C, B2B
- 검진 유형: 일반 검진, 종합검진, 채용검진, 국가검진, 기업검진, 기업검진(국가), 기업검진(특수), 검진확인서
- 예약 상태: 확정대기, 예약확정, 검진완료, 결과등록, 취소요청, 취소완료, 미수검
- 상태 부가 조건: 평점 작성 전/후, 기간만료 전/후, 이전 상태가 확정대기인지 예약확정인지
- 결제 방식: 즉시결제, 현장결제, 카드결제, 계좌이체
- 가격 정책: 멤버십 적용 여부, 멤버십 선택 여부, 멤버십 잔여 횟수, 지원금 여부, 지원금 잔여 여부, 가격노출 여부, 결제금액 0원/1원 이상 여부, 할인금액, 추가비용, 부가세/케어비 노출 여부
- 가격 표시 위치: 목록/상세 등 결제화면 외 영역, 결제화면, 예약내역 카드, 예약내역 상세
- 가격 산식 조건: 상품금액, 마진금액, 추가비용, 케어비, 할인금액, 지원금, 부가세, 지원금 선적용/후적용
- 툴팁 노출: 예상요금, 착한의사 케어비, 부가세, 상품 마진가 없음에 따른 예상요금 툴팁 hide
- 버튼 영역: 예약내역카드 버튼, 예약내역 상세 하단 버튼
- 버튼 종류: 변경, 예약취소, 결제 영수증, 삭제하기, 1:1 주치의 상담, 평점남기기
- 버튼 상태: 없음, 활성, 비활성, 라인 버튼, 강조 버튼, 아이콘 버튼, 2버튼 병렬 노출
- 정책 전환: 노출에서 비노출로 바뀌는 `hide`, 비노출에서 노출로 생기는 `없음/있음`, 기존 주문건/신규 주문건 차이

## 확인 필요

- A-0, A-1 등 라벨의 정확한 행/열 정의는 Figma 표에서 확대 확인이 필요하다.
- `지원금 여부`, `결제방법`, `버튼`의 B2B 조합표는 TC 작성 직전에 Figma 원본 표 기준으로 재확인한다.
- 안내 문구는 문구 자체가 expected result가 될 수 있으므로 실제 TC 작성 시 최신 기획서 문구와 완전 일치 여부를 다시 확인한다.
- 결제정보 신규 주문건과 기존 주문건의 분기 기준은 주문 생성일/결제정보 스키마/API 응답 기준인지 확인이 필요하다.
- 예약내역카드 버튼 정책표의 각 행 이름과 `O`, `X`, `O(disable)`, `X(disable)`의 정확한 의미는 TC 작성 직전에 원본 표 확대 기준으로 재확인한다.
- 예약내역 상세 하단 버튼 정책표의 각 버튼 컬럼명과 `O`, `X`, `O(disable)`의 의미는 TC 작성 직전에 원본 표 확대 기준으로 재확인한다.
- 가격 factor `P`, `m`, `E`, `S`, `D`, `G`, `v`의 정확한 API 필드 매핑과 화면 행 매핑은 TC 작성 전에 확인한다.
- B2C 화면에 보이는 `검진지원금 본인` 라벨의 정확한 정책 의미는 확인이 필요하다.
- B2B 표시 매트릭스의 `A/B/C/D/E` 라벨 전체 의미와 `C-4`, `C-5`, `D-1` 외 항목명은 원본 표 기준으로 재확인한다.
